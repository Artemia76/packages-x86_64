How to build a package

Introduction

We at NuTyX would like to encourage you to customise your own NuTyX LFS system. We have provided the required Know-how, the available base commands, the minimum syntax of instructions and a list of available packages for you to choose from.

Come and join the many participants of all ages, male and female, from all over the world, and venture along this exciting road we call NuTyX. We can promise you a rewarding experience as you build your very first NuTyX system, following our (NuTyX) rules .

Please note that if you are using a version of the cards package manager that is older then 2.0, it's time to update it.

We offer 5 possible installation Scenarios for you to examine.

  1. Scenario A: "I do not want to compile anything. The available binaries are enough for me" . This option may seem irrelevant or unimportant but it does allow you to keep your favorite distribution as lightweight as possible.
  2. Scenario B: "The available binary packages are insufficient. I would like to add a few extra packages".
  3. Scenario C: "I don't want to use the cli and gui binary package collections. I prefer to compile them myself".
  4. Scenario D: "I want to use the available cli and gui binary package collections as dependencies for my personal ports. Only my personal ports should be compiled locally".
  5. Scenario E: "I want to build my ports in an isolated environment. I also want to keep the original system clean and isolated from the development package environment. Later I can install packages that were compiled in my isolated environment into NuTyX".
